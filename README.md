# IdeaTeamFlydev

基于 `ESP32 + MAVLink + WiFi` 的低成本机队编队控制项目，面向多旋翼/固定翼的基础编队飞行实验。

项目目标是让长机和僚机在不依赖地面站电脑的情况下，通过机载 ESP32 建立局域网并交换编队状态，由设备网页完成主要配置与控制。当前方案优先强调“能飞、能调、能排障”，而不是追求厘米级高精度编队。

## 项目特点

- 基于 `ESP32` 与飞控串口通信，使用 `MAVLink` 读写飞控状态与控制指令
- 长机使用 `AP` 模式建热点，僚机使用 `STA` 模式自动连接长机
- 设备自带网页端，支持配置查看、状态诊断和机队实时控制
- 支持 Captive Portal，连接设备 WiFi 后可自动打开网页
- 支持多旋翼自动解锁起飞、自动加入编队、自动返航/降落
- 支持固定翼手动起飞后切入 `GUIDED` 加入编队
- 加入了 UDP 报文认证、防回放、链路诊断、动作原因码等安全与排障能力
- 最近版本加入了“保守跟随”逻辑，用于降低僚机突然乱飞、突然猛冲、突然压低高度的风险

## 适用场景

- 多架无人机编队飞行实验
- 低成本编队控制原型验证
- ESP32 与 ArduPilot/MAVLink 联动实验
- 现场试飞调参与链路排障

## 项目限制

这个项目当前更适合“实验型松编队”，不适合直接理解为高精度近距编队系统。

- 主要依赖 `GPS + 气压计 + 罗盘`
- 没有使用 RTK、UWB、视觉定位、激光测距等高精度相对定位能力
- 编队距离设置为 `1m` 只是软件允许值，不代表普通 GPS 条件下能稳定实现 `1m` 实际间距
- 多旋翼自动加入编队后，依然可能存在几米级横向误差
- 固定翼与多旋翼的控制逻辑不同，固定翼默认更保守

如果你希望做真正贴近的近距编队，请考虑增加 RTK、UWB、视觉或其他相对定位传感器。

## 硬件需求

- ESP32-C3 开发板或 ESP32-S3 开发板
- 支持 MAVLink 的飞控
- 飞控与 ESP32 的 UART 串口连接
- 可正常工作的 GPS、气压计、罗盘
- 稳定供电

当前代码中已对以下板型定义了默认引脚：

- `ARDUINO_ESP32C3_DEV`
- `ARDUINO_ESP32S3_DEV`

如需修改串口引脚，可编辑 `g_head.h` 中的板卡宏定义。

## 软件环境

- Arduino IDE
- ESP32 Arduino Core
- 飞控侧需支持 MAVLink，并具备 `GUIDED / LOITER / RTL / LAND` 等模式语义

本项目当前仓库结构是 Arduino 工程，主入口文件为：

- `IdeaTeamFlydev.ino`

## 默认配置

首次启动或重置配置后，默认值包括：

- 默认模式：长机模式 `AP`
- 默认 WiFi 名称：`IdeaTeamFlyXXXX`
- 默认 WiFi 密码：`12345678`
- 默认编队距离：`1m`
- 默认编队高度偏移：`0`
- 默认多旋翼自动起飞：关闭
- 默认多旋翼自动降落：关闭

其中默认 SSID 会根据网卡 MAC 后两字节自动生成。

## 基本接线

请根据你的 ESP32 板型和飞控串口接口进行连接，核心原则如下：

- 飞控 `TX` -> ESP32 `RX`
- 飞控 `RX` -> ESP32 `TX`
- `GND` 共地
- 供电稳定，避免飞控侧或 ESP32 侧掉电重启

当前默认串口波特率在代码中为：

```cpp
#define BAUD_RATE 115200
```

如果你的飞控串口波特率不同，请同步修改飞控与代码配置。

## 编译与烧录

1. 使用 Arduino IDE 打开 `IdeaTeamFlydev.ino`
2. 安装 ESP32 开发板支持包
3. 选择对应板型
   - `ESP32C3 Dev Module`
   - 或 `ESP32S3 Dev Module`
4. 编译并烧录到 ESP32
5. 打开串口监视器，波特率 `115200`

## 使用流程

### 1. 设置长机

- 烧录后默认进入长机模式
- ESP32 会建立热点
- 使用手机或电脑连接设备 WiFi
- 默认密码为 `12345678`
- 连接后网页会自动弹出，或手动访问设备地址

### 2. 设置僚机

- 在网页中把模式切换为僚机 `STA`
- 填入长机 WiFi 名称和密码
- 保存后重启
- 僚机会自动连接长机并开始收发编队报文

### 3. 网页配置

网页中可配置：

- 模式：长机 / 僚机
- WiFi 名称 / 密码
- 编队距离
- 编队高度偏移
- 编队类型
- 僚机序号
- 多旋翼自动起飞策略
- 多旋翼自动返航/降落策略
- RC 辅助通道同步

### 4. 机队实时控制

长机网页支持以下实时指令：

- 解锁测试
- 上锁测试
- 一键起飞僚机
- 僚机加入编队
- 返航/降落所有僚机

当前版本已去掉控制指令的网页密码验证，仅保留确认框。连接设备 WiFi 即视为已授权进入控制页面。

## 当前控制逻辑说明

### 多旋翼

- 可选自动解锁起飞
- 起飞后会先短暂稳定悬停
- 随后自动加入编队
- 已加入编队后采用保守跟随策略：
  - 目标点渐进收敛
  - 水平移动限速
  - 高度变化限幅
  - 防止目标高度突然压低
- 长机满足条件时可自动返航/降落

### 固定翼

- 默认不支持跟随长机自动起飞
- 需要飞手手动起飞
- 达到安全高度后手动切入 `GUIDED`
- 再加入编队
- 恢复/降落逻辑更接近 `RTL`

## 安全机制

项目当前已经加入以下保护：

- UDP 报文认证
- 报文计数器防回放
- 领航机来源校验
- WiFi 重连窗口与链路状态诊断
- 动作原因码显示
- 飞控 `COMMAND_ACK` 记录
- 长机上锁显式事件广播
- 多旋翼加入编队时的平滑收敛与高度保护

## 重要安全提醒

本项目涉及真实飞行器控制，请务必谨慎使用。

- 首次测试请务必拆桨或固定机体
- 真机试飞请先单机验证，再双机验证
- 先验证解锁、模式切换、起飞、降落、链路恢复，再验证自动编队
- 不要在人员密集区或复杂磁环境中直接试飞
- 普通 GPS 条件下，不建议追求过小编队距离
- 如果出现异常飘飞、异常掉高、模式切换不符预期，应立即中止试飞

建议初期使用以下保守参数：

- 编队距离：`6m ~ 10m`
- 编队高度偏移：`0` 或略高于长机
- 风小、空旷场地、视距内测试

## 调试与排障

串口日志会持续输出：

- 飞控连接状态
- GPS / 高度 / 电池信息
- WiFi 状态
- 领航机链路年龄
- 命令 ACK
- 起飞 / 降落 / 加入编队原因

如果试飞出现异常，优先排查：

- 飞控是否正确进入 `GUIDED`
- GPS 与 EKF 是否健康
- 长机与僚机 WiFi 是否稳定
- 领航机广播高度是否正常
- 僚机的 `relative_alt` 是否异常漂移
- 编队高度偏移是否设置过低或为负值

## 已知问题

- 仅依赖普通 GPS 时，实际编队距离会明显大于理论设置值
- 加入编队仍可能存在米级横向摆动
- 不同飞机气压计与 home 基准不一致时，可能影响相对高度控制
- 固定翼与多旋翼混编时，需要更保守的测试策略
